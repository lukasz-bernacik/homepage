CSS - Cascading Style Sheets

CSS odpowiadają za prezentację. Wpływają na wygląd elementów HTML na ekranie, wydrukach itp. To właśnie CSS jest odpowiedzialny za to jak pod względem wizualnym prezentuje się nasza strona. Kolor czcionki, tła, wielkość czcionki, wygląd list. To tylko część rzeczy za które odpowiedzialny jest css. Pliki zawierające kod CSS mają rozszerzenie .css

Składnia CSS to reguły:

selektor {
właściwość: wartość;
}

Specyficzność

W CSS żeby dostać się do elementu, który chcemy ostylować, możemy się dostać na kilka sposobów. Poprzez element, klasę, id, styl liniowy albo !important. Każdy z tych sposobów ma inną "siłę".

Koleność od najsłabszej reguły:
  1. element
  2. klasa
  3. id
  4. styl liniowy
  5. !important

O klasach

Dobrą praktyką jest dodawanie klasy do wszystkich elementów i unikanie zagnieżdżeń. Nie używamy stylów liniowych i !important ponieważ tworzą bałagan w kodzie i przy dużej stronie możemy się pogubić.

Nazewnictwo klas:
  • używaj pojedynczych słów z j. angielskiego
  • nazywaj rzeczy zgodnie z ich funkcją na stronie
  • startuj z małej litery np. header nie Header
  • nie zczynaj od cyfry
  • jeśli musisz użyć dwóch słów używaj tzw. kebab case np. news-listing
  • staraj się, żeby element miał jedną klasę (nie zwiększaj specyficzności bez potrzeby

CSS stylujemy po:

  • elementach np. h1
  • klasach np. .content
  • id np. #image

Ważne informacje:

  • html class="name" = css .name
  • html id="name" = css #name
  • stylujemy po klasach (bez zagnieżdżeń)
  • nazywamy klasy po angielsku zgodnie z przeznaczniem
  • tworzymy klasy nawet jeśli mamy jeden element danego typu
  • specyficzność może dać popalić
  • nie używamy !important i stylów inline'owych
